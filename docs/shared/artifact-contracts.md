# Artifact Contracts

## Overview

Artifact contracts define the files produced and consumed by each skill in the Olympus system. They specify who writes each file, who reads it, and at which phase it is created. This ensures clean handoffs between agents and skills.

## State Directory

All artifacts are stored under:

```
.olympus/{id}/
```

## ID Format

```
{skill}-{YYYYMMDD}-{short-uuid}
```

The short UUID is generated via `uuidgen` and truncated to 8 characters.

**Examples:**
- `oracle-20250115-a3f8b2c1`
- `pantheon-20250115-7d2e9f04`
- `tribunal-20250116-b1c4d5e6`

## File Format Rules

- `.md` -- Markdown for human-readable documents and analysis outputs
- `.json` -- JSON for structured data, scores, and machine-readable records
- All JSON files must be **pretty-printed** (indented with 2 spaces)

## Oracle Contracts

| File                  | Phase | Writer        | Reader            |
|-----------------------|-------|---------------|-------------------|
| codebase-context.md   | 1     | Orchestrator (from Hermes) | Apollo, Metis     |
| interview-log.md      | 2     | Orchestrator (from Apollo) | Metis             |
| ambiguity-scores.json | 2     | Orchestrator (from Apollo) | Gate check        |
| gap-analysis.md       | 4     | Orchestrator (from Metis) | Zeus, Helios      |
| spec.md               | 5     | Orchestrator  | All downstream    |

### File Descriptions

- **codebase-context.md** -- Summary of the relevant codebase structure, key files, and architectural patterns discovered by Hermes during the reconnaissance phase.
- **interview-log.md** -- Record of the interactive Q&A session between Apollo and the user, capturing requirements, constraints, and acceptance criteria.
- **ambiguity-scores.json** -- Scored ambiguity assessment for goal, constraints, and acceptance criteria. Used for gate checks per [Ambiguity Scoring](ambiguity-scoring.md).
- **gap-analysis.md** -- Metis's analysis of gaps between the stated requirements and the current codebase state.
- **spec.md** -- The final refined specification output by Oracle, consumed by all downstream skills.

## Pantheon Contracts

| File                    | Phase | Writer          | Reader            |
|-------------------------|-------|-----------------|-------------------|
| source-catalog.md       | 0     | Orchestrator    | All agents        |
| source-scope-analyst.md | 0     | Orchestrator    | Analyst agents    |
| source-scope-da.md      | 0     | Orchestrator    | Eris              |
| perspectives.md         | 2     | Orchestrator (from Helios) | All agents        |
| context.md              | 2     | Orchestrator    | All agents        |
| analyst-findings.md     | 3     | Orchestrator (from analysts) | Eris              |
| da-evaluation.md        | 4     | Orchestrator (from Eris) | Consensus stage   |
| prior-iterations.md     | 5     | Orchestrator    | Re-entry          |
| analysis.md             | 5     | Orchestrator    | Downstream        |

### File Descriptions

- **source-catalog.md** -- Full catalog of data sources discovered and selected during the [Source Scope Mapping](source-scope-mapping.md) protocol.
- **source-scope-analyst.md** -- Scope instructions for analyst agents, specifying which sources to consult.
- **source-scope-da.md** -- Scope instructions for Eris, specifying verification checklist for source usage.
- **perspectives.md** -- The list of analytical perspectives generated by Helios, each passing the [Perspective Quality Gate](perspective-quality-gate.md).
- **context.md** -- Shared context document providing problem background, constraints, and codebase summary for all agents.
- **analyst-findings.md** -- Combined findings from all analyst agents, organized by perspective.
- **da-evaluation.md** -- Eris's devil's advocate evaluation of analyst findings, including fallacy detection and evidence verification.
- **prior-iterations.md** -- Summary of previous iteration results, used for feedback loops on re-entry.
- **analysis.md** -- The final consolidated analysis output from Pantheon.

## Genesis Contracts

| File                    | Phase | Writer        | Reader              |
|-------------------------|-------|---------------|---------------------|
| gen-{n}/ontology.json   | 3     | Orchestrator  | Convergence check   |
| gen-{n}/spec.md         | 3     | Orchestrator  | Next generation     |
| gen-{n}/wonder.md       | 1     | Orchestrator (from Metis) | Reflect             |
| gen-{n}/reflect.md      | 2     | Orchestrator (from Eris) | Seed                |
| lineage.json            | 5     | Orchestrator  | Rewind              |
| convergence.json        | 4     | Orchestrator  | Evolution halt      |

### File Descriptions

- **gen-{n}/ontology.json** -- The ontology (concept map) for generation N, used to check convergence between generations.
- **gen-{n}/spec.md** -- The evolved specification for generation N, serving as input for the next generation.
- **gen-{n}/wonder.md** -- Metis's "wonder" phase output: open questions, unexplored angles, and creative provocations for generation N.
- **gen-{n}/reflect.md** -- Eris's reflection on generation N: critique, identified weaknesses, and suggestions for evolution.
- **lineage.json** -- Complete lineage tracking across all generations, recording the evolution path.
- **convergence.json** -- Convergence metrics between consecutive generations, used to determine when evolution should halt.

### File Descriptions

## Odyssey Contracts

| File                        | Phase | Writer        | Reader                  |

| plan.md                     | 4     | Zeus          | Themis, Prometheus      |
| implementation-report.md    | 5     | Prometheus    | Hephaestus, Tribunal    |

### File Descriptions

- **odyssey-state.json** -- Persistent state file for the Odyssey meta-skill. Records the current position in the Oracle -> Pantheon -> Tribunal pipeline, enabling recovery and resumption after interruption.
- **plan.md** -- Implementation plan generated by Zeus containing task breakdown, dependency ordering, AC mappings, and execution sequence. Reviewed by Themis and consumed by Prometheus for implementation.
- **implementation-report.md** -- Prometheus's summary of all code changes made during execution: files modified, files created, test results, and deviations from plan. Consumed by Hephaestus for build verification and by Tribunal for evaluation.

## Review-PR Contracts

| File                    | Phase | Writer        | Reader                              |
|-------------------------|-------|---------------|-------------------------------------|
| pr-diff.patch           | 0     | Orchestrator  | Hermes                              |
| spec-context.md         | 0.5   | Orchestrator  | Helios, Ares, Poseidon, Eris, Nemesis |
| pr-context.md           | 1     | Orchestrator (from Hermes) | Helios, Ares, Poseidon, Eris, Nemesis |
| review-perspectives.md  | 2     | Orchestrator (from Helios) | Ares, Poseidon, dynamic reviewers  |
| review-findings.md      | 3     | Orchestrator (from reviewers) | Eris, Nemesis                    |
| da-evaluation.md        | 4     | Orchestrator (from Eris) | Nemesis                           |
| verdict.md              | 5     | Orchestrator (from Nemesis) | User, GitHub                     |
| review-pr-state.json    | all   | Orchestrator  | State recovery (auto mode)          |

### File Descriptions

- **pr-diff.patch** -- Raw diff of the pull request, fetched from GitHub and saved for reviewer reference.
- **spec-context.md** -- Domain context extracted from existing spec.md/ontology.json, used when spec is available (--spec flag or auto-detected).
- **pr-context.md** -- Hermes's PR reconnaissance: changed files, modules affected, diff summary, and codebase context for reviewers.
- **review-perspectives.md** -- Helios's perspective generation tailored to PR review (code quality, security, domain-specific, performance).
- **review-findings.md** -- Aggregated findings from all reviewer agents (Ares, Poseidon, and any dynamic reviewers), organized by perspective.
- **da-evaluation.md** -- Eris's adversarial challenge of all findings: false positive detection, unsupported claims, missing evidence.
- **verdict.md** -- Nemesis's synthesized verdict: APPROVE/REQUEST_CHANGES/COMMENT with confidence score, deduplicated findings, and reasoning.
- **review-pr-state.json** -- Auto mode state tracking: reviewed PR numbers, commit SHAs, verdicts, and timestamps for deduplication across runs.

## Evolve Contracts

| File                    | Phase | Writer        | Reader            |
|-------------------------|-------|---------------|-------------------|
| benchmark.md            | 1     | Orchestrator  | All phases        |
| dogfood-result.md       | 2     | Orchestrator  | Athena, Metis     |
| eval-matrix.md          | 3     | Orchestrator  | Eris, Metis       |
| diagnosis.md            | 4     | Orchestrator  | Prometheus        |
| refinement-log.md       | 5     | Orchestrator  | Tracking          |
| evolve-state.json       | all   | Orchestrator  | State recovery    |

### File Descriptions

- **benchmark.md** -- Benchmark task definition including target skill, test scenario, expected quality thresholds, and test input data.
- **dogfood-result.md** -- Complete output from running the target skill on the benchmark, including all agent outputs, round counts, gate results, and handoff records.
- **eval-matrix.md** -- Athena's quality evaluation across 5 dimensions (Specificity, Evidence Density, Role Adherence, Efficiency, Actionability) with scores and evidence.
- **diagnosis.md** -- Root cause analysis tracing quality issues back to specific agent prompt sections, with concrete improvement proposals.
- **refinement-log.md** -- Record of all prompt changes applied and rejected per iteration, with rationale.
- **evolve-state.json** -- Persistent state tracking iteration count, score history, convergence status, and benchmark reference.
