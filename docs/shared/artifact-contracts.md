# Artifact Contracts

## Overview

Artifact contracts define the files produced and consumed by each skill in the Olympus system. They specify who writes each file, who reads it, and at which phase it is created. This ensures clean handoffs between agents and skills.

## State Directory

All artifacts are stored under:

```
.olympus/{id}/
```

## ID Format

```
{skill}-{YYYYMMDD}-{short-uuid}
```

The short UUID is generated via `uuidgen` and truncated to 8 characters.

**Examples:**
- `oracle-20250115-a3f8b2c1`
- `pantheon-20250115-7d2e9f04`
- `tribunal-20250116-b1c4d5e6`

## File Format Rules

- `.md` -- Markdown for human-readable documents and analysis outputs
- `.json` -- JSON for structured data, scores, and machine-readable records
- All JSON files must be **pretty-printed** (indented with 2 spaces)

## Oracle Contracts

| File                  | Phase | Writer        | Reader            |
|-----------------------|-------|---------------|-------------------|
| codebase-context.md   | 1     | Hermes        | Apollo, Metis     |
| interview-log.md      | 2     | Apollo        | Metis             |
| ambiguity-scores.json | 2     | Apollo        | Gate check        |
| gap-analysis.md       | 4     | Metis         | Zeus, Helios      |
| spec.md               | 5     | Orchestrator  | All downstream    |

### File Descriptions

- **codebase-context.md** -- Summary of the relevant codebase structure, key files, and architectural patterns discovered by Hermes during the reconnaissance phase.
- **interview-log.md** -- Record of the interactive Q&A session between Apollo and the user, capturing requirements, constraints, and acceptance criteria.
- **ambiguity-scores.json** -- Scored ambiguity assessment for goal, constraints, and acceptance criteria. Used for gate checks per [Ambiguity Scoring](ambiguity-scoring.md).
- **gap-analysis.md** -- Metis's analysis of gaps between the stated requirements and the current codebase state.
- **spec.md** -- The final refined specification output by Oracle, consumed by all downstream skills.

## Pantheon Contracts

| File                    | Phase | Writer          | Reader            |
|-------------------------|-------|-----------------|-------------------|
| ontology-catalog.md     | 0     | Orchestrator    | All agents        |
| ontology-scope-analyst.md | 0   | Orchestrator    | Analyst agents    |
| ontology-scope-da.md    | 0     | Orchestrator    | Eris              |
| perspectives.md         | 2     | Helios          | All agents        |
| context.md              | 2     | Orchestrator    | All agents        |
| analyst-findings.md     | 3     | Analyst agents  | Eris              |
| da-evaluation.md        | 4     | Eris            | Consensus stage   |
| prior-iterations.md     | 5     | Orchestrator    | Re-entry          |
| analysis.md             | 5     | Orchestrator    | Downstream        |

### File Descriptions

- **ontology-catalog.md** -- Full catalog of data sources discovered and selected during the [Ontology Scope Mapping](ontology-scope-mapping.md) protocol.
- **ontology-scope-analyst.md** -- Scope instructions for analyst agents, specifying which sources to consult.
- **ontology-scope-da.md** -- Scope instructions for Eris, specifying verification checklist for source usage.
- **perspectives.md** -- The list of analytical perspectives generated by Helios, each passing the [Perspective Quality Gate](perspective-quality-gate.md).
- **context.md** -- Shared context document providing problem background, constraints, and codebase summary for all agents.
- **analyst-findings.md** -- Combined findings from all analyst agents, organized by perspective.
- **da-evaluation.md** -- Eris's devil's advocate evaluation of analyst findings, including fallacy detection and evidence verification.
- **prior-iterations.md** -- Summary of previous iteration results, used for feedback loops on re-entry.
- **analysis.md** -- The final consolidated analysis output from Pantheon.

## Genesis Contracts

| File                    | Phase | Writer        | Reader              |
|-------------------------|-------|---------------|---------------------|
| gen-{n}/ontology.json   | 3     | Orchestrator  | Convergence check   |
| gen-{n}/spec.md         | 3     | Orchestrator  | Next generation     |
| gen-{n}/wonder.md       | 1     | Metis         | Reflect             |
| gen-{n}/reflect.md      | 2     | Eris          | Seed                |
| lineage.json            | 5     | Orchestrator  | Rewind              |
| convergence.json        | 4     | Orchestrator  | Evolution halt      |

### File Descriptions

- **gen-{n}/ontology.json** -- The ontology (concept map) for generation N, used to check convergence between generations.
- **gen-{n}/spec.md** -- The evolved specification for generation N, serving as input for the next generation.
- **gen-{n}/wonder.md** -- Metis's "wonder" phase output: open questions, unexplored angles, and creative provocations for generation N.
- **gen-{n}/reflect.md** -- Eris's reflection on generation N: critique, identified weaknesses, and suggestions for evolution.
- **lineage.json** -- Complete lineage tracking across all generations, recording the evolution path.
- **convergence.json** -- Convergence metrics between consecutive generations, used to determine when evolution should halt.

## Agora Contracts

| File                    | Phase | Writer        | Reader            |
|-------------------------|-------|---------------|-------------------|
| debate-frame.json       | 1     | Orchestrator  | All committee     |
| committee-positions.md  | 3     | Committee     | Eris, Orchestrator|
| da-challenges.md        | 4     | Eris          | Committee         |
| decision.md             | 5     | Orchestrator  | User              |

### File Descriptions

- **debate-frame.json** -- Structured debate frame containing the question, options with descriptions, and context for committee deliberation.
- **committee-positions.md** -- Each committee member's independent position, preferred option, and supporting arguments from debate rounds.
- **da-challenges.md** -- Eris's devil's advocate challenges against all positions (both approved and rejected), including fallacy detection results.
- **decision.md** -- Final decision document with rationale, committee positions, dissent record, DA challenges, and consensus level.

## Audit Contracts

| File                    | Phase | Writer        | Reader            |
|-------------------------|-------|---------------|-------------------|
| audit-mechanical.json   | 1     | Hephaestus    | Athena            |
| audit-semantic.json     | 2     | Orchestrator  | Orchestrator      |
| audit-report.md         | 3     | Orchestrator  | User              |

### File Descriptions

- **audit-mechanical.json** -- Mechanical validation results including YAML frontmatter validity, cross-reference integrity, file existence checks, and contract agent verification.
- **audit-semantic.json** -- Semantic validation results including permission-role consistency, contract completeness, gate threshold consistency, clarity enforcement scan, and delegation pattern compliance.
- **audit-report.md** -- Human-readable audit report summarizing all violations, warnings, and coverage statistics.
